# Package Hallucinations by Code-Generating LLMs (USENIX Security 2025)

## Summary

This paper investigates a novel and emerging security risk: package hallucinations generated by code-generating large language models (LLMs). When developers use LLMs to generate code, the models sometimes reference non-existent packages. Attackers can exploit this by creating malicious packages with those hallucinated names, leading to a new type of supply chain attack. The authors systematically analyze this phenomenon across multiple LLMs and programming languages.

The study quantifies how often hallucinations occur and examines their characteristics. It shows that hallucinated packages are not rare and often follow predictable naming patterns. The authors also demonstrate real-world exploitation scenarios where attackers register these packages and successfully compromise systems. The paper concludes by suggesting mitigation strategies such as package verification, LLM improvements, and developer awareness.

## Discussion

### What Works

This paper is particularly interesting because it highlights a security risk introduced by AI itself. Unlike traditional vulnerabilities, this issue arises from the behavior of machine learning models rather than software bugs. I found this perspective refreshing and highly relevant, especially given the increasing reliance on AI-assisted coding tools.

One thing I liked is the empirical approach. The authors didn't just speculate — they measured hallucination rates and demonstrated actual attack feasibility. This makes the threat very tangible.

### Limitations

However, I felt that the paper could have included more concrete examples of exploited systems to emphasize real-world impact. One area that could have been improved is the discussion of defenses. While the paper suggests solutions, many of them rely on changes to LLMs or developer behavior, which are difficult to enforce. I would have liked to see more system-level defenses, such as package registry protections or automated validation tools.

### Surprising Observations

A surprising finding was how predictable hallucinations are. This suggests that attackers can systematically exploit LLM behavior, which is concerning. It contradicts the assumption that hallucinations are random and therefore difficult to weaponize — instead, they can be anticipated and targeted.

### Overall Take

Overall, the paper is very well-presented and introduces a new class of vulnerabilities. It raises important questions about the security implications of AI-generated code and sets the stage for future research.
